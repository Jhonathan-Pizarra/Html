Charset...

La especificación HTML5 anima a los desarrolladores web a utilizar el juego de caracteres UTF-8, que cubre casi todos los caracteres y símbolos del mundo. por eso se usa la etiqeuta: meta charset="UTF-8".
Antes se usaban ASCII, ANSI, ISO-8859 y finalmente, UTF-8. Pudes ver más sobre estos en diferencias entre conjuntos de caracteres

Urls

Un localizador uniforme de recursos (URL) se utiliza para direccionar un documento (u otros datos) en la web. Una dirección web como https://www.w3schools.com/html/default.asp sigue estas reglas de sintaxis: scheme://prefix.domain:port/path/filename,
dónde:
  • esquema: define el tipo de servicio de Internet (el más común es http o https)
  • prefijo: define un prefijo de dominio (el valor predeterminado para http es www)
  • dominio: define el nombre de dominio de Internet (como w3schools.com)
  • puerto: define el número de puerto en el host (el valor predeterminado para http es 80)
  • ruta: define una ruta en el servidor (si se omite: el directorio raíz del sitio)
  • nombre de archivo: define el nombre de un documento o recurso
Los esquemas de url más comunes son: http, https, file, ftp.

Codificación de URL

Las URL solo se pueden enviar a través de Internet utilizando el juego de caracteres ASCII.
Si una URL contiene caracteres fuera del conjunto ASCII, la URL debe convertirse.
La codificación de URL convierte los caracteres que no son ASCII en un formato que se puede transmitir a través de Internet.
La codificación de URL reemplaza los caracteres que no son ASCII con un "%" seguido de dígitos hexadecimales.

¿Has visto por ejemplo la API de Whatsapp, cuando escribes Hola! la url cambia con unos simbolos extraños? https://www.w3schools.com/action_page2.php?text=Hola%21 es la traducción
Sin embargo si solo pusiera Hola, la traducción sería más sencilla: https://www.w3schools.com/action_page2.php?text=Hola
puedes hacer más pruebas en codificador de caracteres

HTML vs XHTML

¿Recuerdas a xml?: un lenguaje de marcado donde todos los documentos deben estar marcados correctamente (estar "bien formados").
¿Recuerdas a html?: un lenguaje de marcado que nos permite indicar la estructura de nuestro documento mediante etiquetas.
Bueno, pues XHTML fue desarrollado para hacer html más extensible y flexible para trabajar con otros formatos de datos (como XML).
Los navegadores ignoran los errores en las páginas HTML e intentan mostrar el sitio web incluso si tiene algunos errores en el marcado. Por lo que, XHTML viene con un manejo de errores mucho más estricto.
  • ! DOCTYPE es obligatorio
  • El atributo xmlns de "html" es obligatorio
  • html, head, title y body es obligatorios
  • Los elementos siempre deben estar correctamente anidados
  • Los elementos deben estar siempre cerrados
  • Los elementos deben estar siempre en minúsculas
  • Los nombres de atributos siempre deben estar en minúsculas
  • Los valores de atributo siempre deben estar entrecomillados
  • La minimización de atributos está prohibida
